coucou, je suis chapter-four-suite-child et je suis à l'intérieur de mon parent !

Mon parent possède une propriété "adresse" que je souhaite utiliser dans mon template

Pour la recevoir, je peux utiliser le décorateur @Input() devant la propritée

@Input() adresse: string;

Je vais pouvoir la récupérer, à condition qu'il me la passe.

et idem avec la propriété "code".

pour cela, il indique une directive ['adresse']="adresse" dans ma balise, soit:

'<'app-chapter-four-suite-child [adresse]="adresse" [code]="code">'<'/app-chapter-four-suite-child>'

Bon, faut enveler les simple quote du code ci-dessus ... Mais je récupère bien sa propriété adresse

l'adresse du parent est : {{ adresse }}

Et ça, même si elle change dans le parent !!

Je peux faire de même avec la propriété "code" du parent, mais je peux aussi renommé à l'intérieur de l'enfant:

@Input('nomChezLeParent') nomChezEnfant: leType

avec @Input('code') parentCode: number; je peux afficher la valeur de "code" en utilisant {{'{{'}} parentCode }}

le code postal du parent est : {{ parentCode }}

Mais cette méthode de renommage n'est pas appréciée et peu donner une alerte de type "no-input-rename", vous indiquant qu'il est préférable de ne pas renommer une propriété.

C'est sympa, et assez simple ... mais ne concerne que la direction parent -> enfant.

Dans la suite du chapitre, on va voir comment passer de l'enfant au parent.
