importer bootstrap

Pour ce faciliter la vie, rien de tel que bootstrap. Mais pour l'importer, on ne va pas aller dans le fichier index.html pour y glisser un link vers le fichier css.

C'est faisable, mais ce n'est pas une bonne pratique !

on va plutôt utiliser et pa la même occasion découvrir le fichier angular.json qui est à la racine de notre projet.

Celui-ci définit notre application ... avec ses dépendances, ses options pour l'environnement de développement et l'environnement de production, et les tests de l'application ... bref, tout ou presque.

On va ajouter bootstrap comme dépendance à notre projet

npm install bootstrap --save

On va avoir une dossier bootstrap à l'intérieur du dossier nodes_modules

Dans angular.json, on va voir une propriété "build", sous laquelle on va trouver une propriété "styles" comportant un tableau.

on ajoute, avant styles.scss notre css de bootstrap:

"node_modules/bootstrap/dist/css/bootstrap.css",

on remplace le contenu de header.component.html par une navbar de bootstrap et on peut même enlever le scss qu'on avait fait dans header.component.scss

Mais bootstrap utilise des librairies javascript ... par exemple, là, mon dropdown ne fonctionne pas.

Pour ajouter les librairies js nécessaire, on va s'y prendre de la même manière ...

déjà, ajoutons le bottstrap.js dans le fichier angular.json

attention, il ne s'agit pas d'un style, mais d'un script.

sous notre tableau styles, auquel on a ajouté le css de bootstrap, on voit un tableau scripts qui est vide. Ajoutons "node_modules/bootstrap/dist/js/bootstrap.js" à ce tableau !

Il faut aussi popper.js et jquery pour que cela fonctionne ...

npm install popper.js --save
npm install jquery --save

avant de déclarer bootstrap.js, on ajouter jquery et popper:

"scripts": [ "node_modules/jquery/dist/jquery.min.js",
"node_modules/popper.js/dist/umd/popper.min.js",
"node_modules/bootstrap/dist/js/bootstrap.min.js" ]

Je modifie également le footer, avec un stiky footer de bootstrap.

Et vu que bootstrap et disponible partout dans l'application, on va ajouter un peu de style dans app.component.html, en mettant simplement une balise main avec la classe container-fluid autour de ma page et mon router-outlet

D'ailleurs, il serait temps d'en parler de ce router-outlet ... ben allons au chapitre 3 !
